1. Порядок выполнения конструкторов при создании объекта подкласса:

* Конструкторы выполняются строго от родителя к потомку. 

* Сначала вызывается конструктор самого верхнего класса в иерархии (обычно Object). 

* Затем последовательно конструкторы всех промежуточных классов, и только в самом конце — конструктор создаваемого подкласса. 

* Если в конструкторе подкласса явно не указан вызов super, компилятор автоматически вставляет вызов конструктора родителя без параметров.

2. Переопределение static метода и скрытие метода:

* Статические методы переопределять нельзя. 

* Если подкласс объявляет статический метод с такой же сигнатурой, как в родителе, это называется скрытием метода. 

* При скрытии вызываемый метод определяется не типом объекта во время выполнения, а типом ссылки на этапе компиляции. 

* Это принципиально отличается от обычных методов, где работает полиморфизм и вызывается метод фактического объекта.

3. Switch без default при отсутствии совпадений:

* Если ни один case не совпал, а default отсутствует, ничего не происходит — ни один блок кода не выполняется. 

* Программа просто продолжает работу после конструкции switch. Ошибки или исключения не возникает.

4. Невозможность удаления в for-each и правильные способы:

* В for-each нельзя удалять элементы, потому что цикл использует неявный итератор, но не предоставляет доступа к его методу remove. 

* Прямое удаление из коллекции во время итерации приводит к ConcurrentModificationException. 

Правильные способы:

* использовать обычный цикл с явным итератором и его методом remove.

* применить метод removeIf (доступен с Java 8) .

* отфильтровать коллекцию через Stream API с сохранением результата в новую коллекцию.

5. break с меткой vs обычный break:
   
* Обычный break выходит только из самого внутреннего цикла или блока switch, в котором находится. 

* break с меткой может выйти из любого помеченного внешнего блока, включая несколько вложенных циклов сразу. 

* Метка ставится перед блоком кода с двоеточием, а break указывает эту метку.

6. Аннотация Override:
   
* Аннотация Override указывает, что метод должен переопределять метод суперкласса или реализовывать метод интерфейса. 

* Она позволяет компилятору проверить правильность переопределения. 

* Если метод с этой аннотацией фактически ничего не переопределяет (например, из-за опечатки в имени или несовпадения параметров), компилятор выдаст ошибку. 

* Это защита от ошибок при изменении сигнатуры родительского метода.

7. Модификатор для видимости в подклассах:

* Для поля, которое должно быть видимо в подклассах, но скрыто от остального кода вне пакета, используется модификатор protected. 

* Он обеспечивает видимость внутри пакета (как package-private) и дополнительно — во всех подклассах, даже если они находятся в других пакетах.

8. final класс vs sealed класс
   
* final класс вообще нельзя наследовать — у него не может быть подклассов. 

* sealed класс, напротив, можно наследовать, но только строго определённому списку классов, которые перечислены в разрешении permits. 

* Таким образом, final полностью блокирует расширение, а sealed даёт контролируемое, ограниченное расширение. 

* В Java 17 sealed-классы позволяют точнее описывать иерархии, чем просто запрет наследования через final.

9. Несколько конструкторов в классе

* В Java класс может иметь несколько конструкторов — это называется перегрузкой конструкторов. 

* Они различаются количеством, типами и порядком параметров. 

* Один конструктор может вызывать другой с помощью this(...), но такой вызов должен быть первой инструкцией в конструкторе. 

* Наличие нескольких конструкторов даёт гибкость при создании объектов с разными наборами начальных данных.

10. Ковариантный тип возвращаемого значения

* Ковариантный возвращаемый тип — это возможность при переопределении метода изменить тип возвращаемого значения на более конкретный (узкий) подтип исходного типа. 

* Например, если родительский метод возвращает Animal, подкласс может вернуть Dog, который является подклассом Animal. 

* Это безопасно, так как код, ожидающий Animal, всегда сможет работать с Dog. Такое возможно только для ссылочных типов, для примитивов не работает.
